快速安装使用：
    ~ $ gem install jekyll
    ~ $ jekyll new myblog
    ~ $ cd myblog
    ~/myblog $ jekyll serve
    # => Now browse to http://localhost:4000
    
常用命令：
    jekyll build [--watch]
    jekyll serve [--detach] [--watch]

Front-matter：
    Front-matter通过YAML来定义。任何文件都可以包含YAML，但是必须在文件的开头，并用---包含起来。例如，
        ---
        layout: post
        title: Blogging Like a Hacker
        ---
    可用于定义一些变量，预定义的变量如下：
        layout          指定_layouts下文件作为布局，无需文件后缀名
        permalink       指定文章链接名样式，默认为/year/month/day/title.html
        published       设为false可不发布该文章
        category
        categories
        tags
    自定义变量使用方法，如YAML中定义了title，引用时如下，
        {{ page.title }}

发布的blog的格式(Posts)：
    文件名格式：YYYY-MM-DD-title.MARKUP
    内容格式：
        必须以 YAML 格式开始，后面内容可以用 Markdown 或 Textile 格式。
    图片、资源引用方式（以Markdown为例）：
        ![My helpful screenshot]({{ site.url }}/assets/screenshot.jpg)
        … you can [get the PDF]({{ site.url }}/assets/mydoc.pdf) directly.
    目录索引：
        <ul>
          {% for post in site.posts %}
            <li>
              <a href="{{ post.url }}">{{ post.title }}</a>
            </li>
          {% endfor %}
        </ul>
    摘要分隔符：
        excerpt_separator，通过{{ post.excerpt }}引用摘要内容
    代码高亮：
        实例如下，
        {% highlight ruby %}
        def show
          @widget = Widget(params[:id])
          respond_to do |format|
            format.html # show.html.erb
            format.json { render json: @widget }
          end
        end
        {% endhighlight %}

创建其他网页(Pages)：
    有两种方式，直接添加html文件和创建带index.html文件的目录。
    .
    |-- _config.yml
    |-- _includes/
    |-- _layouts/
    |-- _posts/
    |-- _site/
    |-- about.html    # => http://example.com/about.html
    |-- index.html    # => http://example.com/
    └── contact.html  # => http://example.com/contact.html

    .

    ├── _includes/

Data Files:
    位于_data目录下的YAML文件，用于定义通用模板。例如：
        _data/members.yml内容:
            - name: Tom Preston-Werner
              github: mojombo

            - name: Parker Moore
              github: parkr

            - name: Liu Fengyun
              github: liufengyun
        在post或page中使用如下，members.yml的文件名即定义了data下的变量名：
            <ul>
            {% for member in site.data.members %}
              <li>
                <a href="https://github.com/{{ member.github }}">
                  {{ member.name }}
                </a>
              </li>
            {% endfor %}
            </ul>

Liquid Templates
    Filters：提供很多Filter，简单举几个例子如下，
        {{ site.time | date_to_string }}
        {{ “'foo, bar \\baz?'” | uri_escape }} 
        {{ page.content | number_of_words }}
        {{ page.tags | array_to_sentence_string }}
    Tags：几个好用的Tags如下：
        Includes：包含位于_includes目录下的文件。
            {% include footer.html %}
            传参数：{% include footer.html param="value" %}
            参数使用：{{ include.param }}
        代码高亮：Python installed, and set pygments to true in _config.yml file. The default is true.
            {% highlight ruby %}
            {% endhighlight %}
            显示行数：
            {% highlight ruby linenos %}
            {% endhighlight %}
        Post URL：发布位于_posts目录及其子目录下的blog，无需文件后缀
            _posts目录下    {% post_url 2010-07-21-name-of-post %}
            _posts子目录下  {% post_url /subdir/2010-07-21-name-of-post %}
            链接形式        [Name of Link]({% post_url 2010-07-21-name-of-post %})
        gist：
            {% gist 5555251 %}
            {% gist 5555251 result.md %} 制定显示文件名为result.md
            {% gist parkr/931c1c8d465a04042403 %} 私有gist

Permalinks：
    可以在 _config.yml 或 Front-matter 中配置，默认为date。语法如下：
        permalink:     date
    PERMALINK SETTING                           RESULTING PERMALINK URL
    None specified, or permalink: date          /2009/04/29/slap-chop.html
    permalink: pretty                           /2009/04/29/slap-chop/index.html
    permalink: /:month-:day-:year/:title.html   /04-29-2009/slap-chop.html
    permalink: /blog/:year/:month/:day/:title   /blog/2009/04/29/slap-chop/index.html
    permalink: none                             /:categories/:title.html
